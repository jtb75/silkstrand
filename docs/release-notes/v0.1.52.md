# v0.1.45 through v0.1.52

Covers the full arc from pre-refactor agent fixes through the asset-first
refactor and its post-merge stabilisation patches.

## v0.1.45 — nuclei flag fix

- **fix(agent):** Remove deprecated `--severity` flag; use
  `--exclude-severity` inversion so nuclei 3.x runs cleanly.

## v0.1.46 — httpx timeout

- **fix(agent):** Set explicit httpx timeout (`-timeout 10`) to prevent
  the recon pipeline from hanging on unresponsive hosts.

## v0.1.47 — naabu scan mode

- **feat(agent):** Support `SILKSTRAND_NAABU_SCAN_TYPE` env var for
  connect-scan mode (`c`) in unprivileged Docker containers where raw
  sockets are unavailable.

## v0.1.48 — Docker installer

- **feat(agent):** Docker-mode agent installer shell script. Generates a
  `docker run` one-liner with correct mounts, env vars, and restart
  policy. Shipped alongside the install-token flow.

## v0.1.49 — Asset-first refactor

The big one. Full platform refactor from target-first to asset-first
model (ADR 006 + ADR 007). See `docs/release-notes/v0.1.49.md` for the
detailed changelog. Highlights:

- Greenfield migration 017 drops old recon pipeline; creates `assets`,
  `asset_endpoints`, `collections`, `findings`, `scan_definitions`,
  `credential_mappings`, `asset_relationships`.
- Migration 018 restores `agent_allowlists` (dropped in 017, needed for
  UI allowlist viewer).
- Dashboard with KPI tiles, Suggested Actions, Recent Activity widgets.
- Assets 3-tab view, detail drawer, coverage column, bulk actions.
- Collections replace Asset Sets with scope (asset / endpoint / finding).
- Scan definitions + in-process scheduler (cron-driven).
- Unified findings table (network_vuln, network_compliance,
  bundle_compliance).
- Consolidated Credentials page (sources + mappings).
- Correlation rules fire on discovery with `{match: {collection_id}}`
  body shape.

## v0.1.50 — allowlist snapshot_hash fix

- **fix(api):** The `snapshot_hash` field was not being written during
  agent allowlist ingestion, causing stale-snapshot detection to skip.
  Fixed to compute and persist the SHA-256 hash on every
  `allowlist_snapshot` message.

## v0.1.51 — nginx SPA fallback fix

- **fix(web):** nginx `try_files` directive returned 403 instead of
  falling back to `index.html` on deep-link refresh (e.g.
  `/collections/abc`). Root cause: missing `$uri/` check before the
  fallback. Fixed in the tenant-web Dockerfile nginx config.

## v0.1.52 — asset API + rendering fixes

- **fix(api):** Flatten the asset API response — endpoints and findings
  were nested an extra level, causing the frontend to render empty lists.
- **fix(web):** Assets and Endpoints tab rendering corrected to consume
  the flattened response shape.
- **fix(web):** Date formatting utility (`time.ts`) now handles
  edge cases (null, empty string, epoch zero) gracefully instead of
  rendering "Invalid Date".
