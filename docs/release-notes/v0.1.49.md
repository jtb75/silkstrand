# v0.1.49 — Asset-first refactor

## Headline

Full platform refactor from target-first to asset-first model. Nav, data
model, and workflows rebuilt around assets / endpoints / findings /
collections / scan_definitions. Greenfield: no migration path from
v0.1.48; stage was reseeded clean. Prod stays on v0.1.48 until P6
integration smoke completes and we tag v0.1.49 for promotion.

Design of record: [ADR 006](../adr/006-asset-first-data-model.md) and
[ADR 007](../adr/007-findings-scheduler.md). Execution plan:
`docs/plans/asset-first-execution.md`. UI: `docs/plans/ui-shape.md`.

## What's new

- **Assets 3-tab view + detail drawer + coverage column + bulk actions.**
  Pages split Assets / Endpoints / Services with a unified filter bar,
  credential-coverage column, and a detail drawer showing endpoints +
  recent findings.
  (`f24a846` feat(p4-frontend): Collections + Assets 3-tab + detail drawer)
- **Collections replace Asset Sets**, with `scope ∈ {asset, endpoint,
  finding}` — used as the scoping surface for scan_definitions, rules,
  and findings queries. Coverage roll-ups computed server-side.
  (`2bcf8d7` feat(p4-backend): collections finding-scope + rules CRUD +
  coverage roll-ups)
- **Scan definitions + in-process scheduler.** `scan_definitions` CRUD,
  cron-driven scheduler, `execute` / `enable` / `disable`, and a
  `coverage` endpoint showing matched endpoints and credential coverage.
  (`ddd06a4` feat(p3-backend): findings ingest + scan_definitions +
  scheduler; `ce9c03f` feat(p3-frontend): Scans reshape + Findings page)
- **Unified findings table.** `network_vuln`, `network_compliance`, and
  `bundle_compliance` all land in `findings` with a shared suppress /
  reopen lifecycle and a single `/api/v1/findings` surface.
  (`ddd06a4`, `ce9c03f`)
- **Consolidated Credentials page.** `credential_sources` +
  `credential_mappings` unified under a single Credentials UI;
  notification-channel CRUD folded into the same consolidated surface.
  (`8b0731c` feat(p5-b): consolidated Credentials page + mappings)
- **Dashboard** with KPI tiles, Suggested Actions, and Recent Activity
  widgets, backed by `/api/v1/dashboard/{kpis,suggested-actions,
  recent-activity}`.
  (`a28b643` feat(p5-a): dashboard KPIs + widgets + Suggested Actions)
- **Rules fire on discovery.** `EvaluateAsset` is wired into the ingest
  path; rule body shape is now `{match: {collection_id}, actions: [...]}`.
  (`f29c687` feat(p2): ingest + rules engine for asset-first refactor)
- **UUID randomness invariant + guard.** Migration 017 rewrites
  legitimately-leaked dev-seed UUIDs to random v4 and preserves the
  reserved discovery bundle (`11111111-…`). Ordering hardened in
  `edbf244` fix(p1): reorder migration 017 + revert auto-heal.
  (`f29c687`, `edbf244`, `1941761`, `b4a6deb` feat(p6-prep): close
  asset-first audit gaps)
- **Docker-mode agent installer** (shipped earlier in v0.1.48, worth
  calling out for this cycle's onboarding story).

## Breaking changes

- `/api/v1/asset-sets`, `/api/v1/notification-channels`, and
  `/api/v1/one-shot-scans` **removed**. Use `/api/v1/collections`,
  `/api/v1/credential-sources` (channels folded in), and
  `/api/v1/scan-definitions` + `/execute` respectively.
- **Correlation-rule body shape** changed: `{match: {collection_id},
  actions: [...]}`. Old predicate-style `match` no longer accepted.
- **`scan_results` table dropped** — query `/api/v1/findings` instead.
  Old `scans` table rebuilt; `scans.target_id` now nullable and scan
  runs link to `scan_definitions` where applicable.
- **Tenants on stage were reseeded.** No v0.1.48 → v0.1.49 migration
  path; new tenants must go through the current provisioning flow.
- Legacy pages `Scans`, `ScanResults`, `OneShotScans` remain as thin
  compat shims pending removal.

## Known deferrals

- **Email + PagerDuty notification senders** stubbed (webhook + Slack
  are live; HMAC-signed webhooks, secrets encrypted at rest).
- **`suggest_target` rule action logs only**; no DB writeback yet.
- **Notification retry worker** not yet built — failed
  `notification_deliveries` rows stay failed; rule must re-fire.
- **Vault / CyberArk / AWS Secrets Manager resolvers** (ADR 004 C1+)
  not started; `credential_sources.type` still `static` in practice.
- **AWS cloud discovery** (R2) and DNS zone enumeration planned but
  not started.

## Upgrade notes

None — greenfield. Prod stays on v0.1.48 until we tag v0.1.49 to prod
after P6 integration smoke on stage.
